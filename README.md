# 一、项目本质（核心认知）

反诈虾不是一个“AI问答工具”，而是一个：

👉 **用善意行为驱动安全行为的机制型产品**

核心逻辑：

* 不是直接给用户免费AI
* 而是：

👉 做一件好事 → 获得使用AI的资格

本质升级：

| 普通产品  | 反诈虾    |
| ----- | ------ |
| 工具    | 行为系统   |
| 被动使用  | 主动引导   |
| 一次性价值 | 长期行为塑造 |

---

# 二、整体系统结构

系统 = 两端 + 积分系统 + 数据沉淀

## 1️⃣ 善意端（生产分）

用户提交善意 → AI判断 → 给分

## 2️⃣ 使用端（消耗分）

用户咨询 → AI判断诈骗 → 扣分

## 3️⃣ 数据层

* cases.jsonl（诈骗案例）
* good_deeds.jsonl（善意记录）

---

# 三、完整链路（最关键）

用户进入 → 登录 → 主页

分两条路径：

### 善意端：

输入善意 → AI判断 → 加分 → 存储

### 使用端：

输入/图片 → OCR → AI判断 → 扣分 → 存储

👉 形成闭环：

善意 → 分数 → 使用AI → 防被骗 → 再传播善意
